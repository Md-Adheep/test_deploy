What is Amazon EC2?

Elastic Compute Cloud — scalable virtual servers in the cloud

💡

Amazon EC2 (Elastic Compute Cloud) provides resizable compute capacity in the cloud. Think of it as renting a virtual computer from AWS — you choose the CPU, RAM, storage, and OS you need, pay for what you use, and can scale instantly.

Launching an EC2 instance takes just a few minutes through the AWS Console or CLI. Follow these steps to get your virtual server running.

  • 1

    Choose an AMI (Amazon Machine Image)

    An AMI is a template containing the OS and pre-installed software. Select from Amazon Linux 2, Ubuntu 22.04, Windows Server, or custom AMIs from the marketplace.

  • 2

    Select an instance type

    Choose your compute size based on CPU, RAM, and network needs. Start with t3.micro for testing — it's free tier eligible.

  • 3

    Configure network & subnet

    Place your instance inside a VPC and choose a subnet. Use a public subnet for internet access or a private subnet for internal workloads.

  • 4

    Attach a security group

    Security groups act as virtual firewalls. Open port 22 for SSH (Linux) or 3389 for RDP (Windows). Restrict SSH to your IP only!

  • 5

    Create or select a key pair

    A key pair authenticates your SSH connection. Download the .pem file and keep it safe — AWS won't let you download it again.

  • 6

    Launch and connect

    Click Launch. Once running, connect via: ssh -i key.pem ec2-user@<public-ip>

💿

AMI — Amazon Machine Image

Pre-configured snapshot used to launch instances. Contains OS, libraries, and app configs.

🛡️

Security groups

Stateful virtual firewalls controlling inbound and outbound traffic at the instance level.

💾

EBS — Elastic Block Store

Persistent block storage volumes that attach to EC2 like a hard drive. Survives instance stop.

🌐

Elastic IP

A static public IPv4 address you can reassign between instances for fault-tolerant architectures.

🏗️

Availability zones

Isolated data centers within a region. Deploy across multiple AZs for high availability.

🔑

IAM roles

Grant EC2 instances permission to access other AWS services without embedding credentials.
